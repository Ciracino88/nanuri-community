# 디자인 룰

원티드 디자인 시스템 기반 전면 개편. 작업하면서 쌓는 문서라 **결정된 것만** 적혀 있습니다.
빈 항목은 아직 안 정한 것이지 빠뜨린 게 아닙니다.

토큰 정의는 [src/index.css](../src/index.css)의 `@theme` 블록입니다.

## 출처와 신뢰도

토큰은 Figma 변수를 **직독한 원본값**입니다 — 눈대중이나 재해석이 아닙니다.
출처: `figma.com/design/nComAXSHh0jj022PMtT6Du` (Wanted Design System, Community)

**그런데 원본이 전부는 아닙니다.** `get_variable_defs`는 *그 노드가 참조하는 변수만* 돌려주므로,
지금 가진 건 인풋 명세 보드(`16215:30100`)가 쓰는 **부분집합**입니다. 그래서 값마다 출처가 다릅니다.

| 구분 | 항목 |
| --- | --- |
| **원티드 확인값** — 고치지 말 것 | 색 시맨틱 전부, 타입 9단계(크기·행간·자간), 그림자 5단계, 반경 14 |
| **우리가 정한 값** — Figma 를 근거로 대지 말 것 | 반경 16·20, 카드 구성, 화면 레이아웃 |
| **원티드가 발행하지 않음** | 반경 스케일, 간격 스케일 |

값을 바꿔야 하면 어느 칸인지 먼저 보세요. 첫 칸이면 Figma 를 확인하고, 둘째 칸이면 그냥 정하면 됩니다.

## 색

**Primary `#0066FF` = 상호작용 전용.** 버튼·선택 상태·링크. 누를 수 있는 것에만.
옛 퍼플 액센트(`#8b3dff`)를 대체합니다. 퍼플은 `accent-violet`으로 장식 자리에만 남습니다.

**글자 위계는 알파 오버레이입니다.** 이게 옛 체계와 가장 크게 다른 점입니다.

불투명 회색이 아니라 **반투명이라 밑에 깔린 면이 비칩니다.** 흰 배경에서 잰 대비가
회색 면 위에서 그대로 유지되지 않습니다. 어두운 면에 얹을 땐 `static-white`를 쓰세요.

아래 대비는 **실제로 합성해서 측정한 값**입니다 (흰 면 `#FFFFFF` / 캔버스 `#F7F7F8` 기준).

| 토큰 | 값 | 합성색 | 흰 면 | 캔버스 | 쓰는 자리 |
| --- | --- | --- | --- | --- | --- |
| `label-normal` | `#171719` | `#171719` | 17.90 | 16.72 | 제목·본문 |
| `label-neutral` | `#2E2F33` 88% | `#47484B` | 9.14 | 8.54 | **보조 텍스트는 여기까지** |
| `label-alternative` | `#37383C` 61% | `#858588` | **3.68** | **3.44** | ⚠ 아래 참고 |
| `label-assistive` | `#37383C` 28% | `#C7C8C9` | 1.68 | 1.56 | 장식·비활성 — **본문 금지** |
| `label-disable` | `#37383C` 16% | — | — | — | 비활성 |
| `primary-normal` | `#0066FF` | `#0066FF` | 4.83 | 4.51 | 상호작용 |

### ⚠ `label-alternative` 는 본문 대비 미달입니다

**3.68:1 로 WCAG AA(4.5:1)에 못 미칩니다.** 18.66px 굵은 글자나 24px 이상에서만 통과합니다.
원티드 원본값이라 우리가 틀린 게 아니지만, 13px 날짜·장소 같은 자리에 쓰면 **옛 `fg-muted`
(4.5:1 통과)보다 읽기 나빠집니다.** 리디자인이 가독성을 후퇴시키는 셈입니다.

**읽어야 하는 보조 텍스트는 `label-neutral`(9.14)을 쓰세요.** `label-alternative` 는 큰 글자,
또는 안 읽혀도 되는 자리에만 씁니다.

`primary-normal` 도 흰 글자 대비 4.83:1 로 아슬아슬합니다. 작은 글자를 파랑 위에 얹을 땐
굵기를 `font-semibold` 이상으로 두세요.

### 그 밖의 대비 미달 (측정값)

| 항목 | 흰 면 대비 | 기준 | 대응 |
| --- | --- | --- | --- |
| `status-negative` `#FF4242` | 3.44 | 본문 4.5 | 에러는 **색만으로 알리지 않는다** — 테두리 + (!) 아이콘 + 문구 셋을 같이 쓴다 |
| `status-positive` `#00BF40` | 2.46 | 본문 4.5 | **글자색으로 쓰지 말 것.** 배경·아이콘·점 표시로만 |
| `line-solid` `#EAEBEC` | 1.19 | UI 요소 3.0 | 인풋 경계가 거의 안 보인다. 아래 참고 |

셋 다 원티드 원본값이라 **토큰을 고치지 않았습니다.** 대신 쓰는 방식으로 우회합니다.

`TextField` 의 에러 표시가 그 예입니다 — 빨간 테두리(3.44)만 믿지 않고 (!) 아이콘과 문구를
같이 띄웁니다. 색각 이상 사용자에게 색은 신호가 아니기 때문이고, WCAG 1.4.1(색에만 의존 금지)이
요구하는 바이기도 합니다.

**`line-solid` 1.19 는 미해결입니다.** 인풋 테두리가 흰 면 위에서 거의 안 보입니다(WCAG 1.4.11 은
UI 요소 경계에 3:1 을 요구). 원티드 명세 자체가 그렇게 생겼고, 인풋을 캔버스(`bg-alternative`)
위에 놓으면 조금 나아지지만 근본 해결은 아닙니다. 폼 화면(비용 청구·프로필 설정)을 그릴 때
다시 판단해야 합니다.

면과 선:

- `bg-alternative` `#F7F7F8` — 페이지 캔버스
- `bg-normal` `#FFFFFF` — 카드·시트 면
- `line-solid` `#EAEBEC` — 불투명 구분선. **선이 겹치는 자리엔 이걸 쓰세요**
- `line-normal`/`neutral`/`alternative` — 알파라 겹치면 진해집니다

상태:

- `status-positive` `#00BF40` / `status-negative` `#FF4242`
- `status-bg-active` `#EAF2FE` — 진행·모집 중 배지 배경
- `status-bg-idle` `#DBDCDF` — 종료·비활성 배지 배경

## 타입

Pretendard. 원티드 파일은 다국어용 `Pretendard JP`를 쓰지만 한국어 전용인 이 앱은
기본 `Pretendard`를 씁니다 — 메트릭이 같아 디자인이 어긋나지 않고 일본어 글리프만 빠집니다.

`text-caption1`(12) · `label2`(13) · `label1`(14) · `body2`(15) · `body1`(16) ·
`body1-reading`(16, 행간만 넓음) · `headline2`(17) · `title3`(24) · `title1`(36)

**자간이 크기별로 다른 게 이 시스템의 핵심입니다.** 큰 글자는 좁히고(−2.7%) 작은 글자는
넓힙니다(+2.52%). 광학 보정이라 빼면 원티드다운 인상이 죽습니다. 스케일에 이미 박혀 있으니
`tracking-*`으로 덮어쓰지 마세요.

**굵기는 스케일에 없습니다** — `font-medium`/`font-semibold`/`font-bold`로 따로 줍니다.
원티드의 "Bold"는 본문·라벨에선 SemiBold(600), 제목에선 Bold(700)입니다.

**입력 필드는 `text-body1`(16px) 밑으로 내리지 마세요.** iOS Safari 가 포커스 시 확대합니다.

## 간격

**토큰을 만들지 않습니다.** Tailwind 기본 단위가 이미 4px 그리드(`--spacing: 0.25rem`)이고
원티드가 쓰는 값도 전부 4의 배수라 어긋나지 않습니다. 여기에 별도 스케일을 얹으면
두 체계가 공존해서 오히려 해롭습니다. `p-4`(16) · `gap-3`(12) 처럼 기본 클래스를 씁니다.

## 반경

원티드가 스케일을 발행하지 않아 확인값은 **14 하나뿐**입니다.

- `rounded-field` (14) — 인풋·버튼. **원티드 확인값**
- `rounded-card` (16) — 카드·리스트 아이템. 우리 값
- `rounded-sheet` (20) — 바텀시트·모달. 우리 값
- `rounded-full` — 칩·배지·아바타

원티드는 큰 라운드를 쓰는 시스템이 아닙니다. 옛 디자인의 20/24 를 그대로 가져오면
원티드 인상이 안 납니다. 전반적으로 절제하세요.

## 그림자

`shadow-xsmall` → `small` → `medium` → `large` → `xlarge`. 전부 원티드 원본값입니다.
면은 테두리가 아니라 그림자로 띄워 분리합니다. 카드는 `shadow-small`이 기본입니다.

## 하단 탭바

**떠 있는 글래스 캡슐**입니다. 인스타그램처럼 화면 하단에 붙지 않고 살짝 띄웁니다.

아직 구현 전이지만 **레이아웃에 지금 영향을 줍니다** — 캡슐이 콘텐츠 위에 뜨므로,
화면 하단에 캡슐 높이 + 여유만큼 패딩을 미리 비워둬야 마지막 항목이 캡슐에 가리지 않습니다.
옛 `sticky` 탭바는 자리를 차지했기 때문에 이 패딩이 필요 없었습니다.

탭은 셋입니다: **소모임 · 찬양팀 · 내정보**.
홈은 소모임에 흡수됐고(행사 카드가 소모임 화면 상단으로), 갤러리는 삭제, 관리자는 내정보에서 진입합니다.

## 마이그레이션 상태

`@theme` 안에 **폐기 예정 블록**으로 옛 토큰이 남아 있습니다. 화면을 새로 그릴 때마다
하나씩 걷어내고 마지막 페이지가 끝나면 블록째 지웁니다.

⚠ **토큰을 지워도 빌드는 통과합니다.** Tailwind 가 클래스를 조용히 안 만들 뿐이라
화면이 무스타일로 뜹니다. 그래서 한 번에 못 지우고 페이지 단위로 옮깁니다.

`@theme static`인 이유: 기본값(트리셰이킹)이면 클래스로 안 쓰인 토큰이 CSS 에서 사라져,
인라인 `style`·SVG `fill` 에서 `var(--color-...)` 로 참조할 때 빈 값이 됩니다.

### 화면 진행

| 화면·컴포넌트 | 상태 |
| --- | --- |
| 소모임 (행사 흡수) | **완료** — 옛 토큰 0건 |
| 소모임 개설 시트 | **완료** |
| `ui/` 프리미티브 — Button · TextField · TextArea · SelectField · BottomSheet | **완료** |
| `ui/ActionRow` | **완료** — `tint` prop 폐기 |
| 찬양팀 · 내정보 · 행사 3종 · 비용 청구 · 관리자 · 프로필 설정 · 게이트 · 로그인 | 대기 |

프리미티브가 먼저 넘어갔으므로, 아직 안 그린 페이지에도 **원티드 인풋·버튼이 이미 뜹니다.**
옛 화면 안에 새 컴포넌트가 섞여 보이는 건 버그가 아니라 이 순서 때문입니다.

## 카테고리 색은 없습니다

**앱에서 카테고리 색을 전부 걷어냈습니다.** 원티드에는 카테고리 팔레트가 없고 액센트도
violet/cyan 둘뿐이라, 항목마다 색을 돌리면 원티드 인상이 깨집니다.

- 아이콘 타일은 `bg-status-bg-active` + `text-primary-normal` **하나뿐**입니다
- 순번 원(행사 순서)도 단색입니다 — 순번은 **위치일 뿐 의미가 아니라서** 색을 나눌 이유가 없습니다
- 행사마다 색을 배정하던 `colorForEvent(id)` 도 없앴습니다

지워진 것: `constants/tints.ts`, `lib/eventColor.ts`, `theme.ts` 의 `TINT_COLORS`/`TAB_COLORS`,
`@theme` 의 teal/pink/amber 토큰, `ActionRow` 의 `tint` prop.

옛 토큰이 남았는지 세는 법:

```bash
grep -rn "text-fg\|bg-card\|bg-surface\|bg-sunken\|text-accent\|bg-accent\|border-line\b\|rounded-tile\|rounded-panel\|shadow-card\|shadow-lift\|shadow-accent\|text-caption\b\|text-body\b\|text-heading\|text-micro\|-subtle\|-strong\b" src/
```
